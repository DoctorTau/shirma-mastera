# Impeccable Design Context — Грифель (Grifel)

## Design Context

### Users
Dungeon Masters running tabletop D&D 5e sessions in Russian, primarily on an iPad propped at the table during play. They need fast, glanceable access mid-session: tracking initiative and HP in combat, looking up monster stat blocks (sourced from dnd.su), and prepping encounters between sessions. Hands are often busy with dice/minis, so touch targets and legibility at a glance matter more than density.

### Brand Personality
Antique grimoire / DM's leather-bound tome. Three words: **ceremonial, weathered, precise**. It should feel like a hand-bound spellbook with brass-and-leather hardware, not a generic productivity app — ritualistic without being kitschy, legible without being clinical.

### Aesthetic Direction
Established by a reference mockup (`D&D Companion (standalone).html`, decoded to `/tmp/dnd_template_decoded.html`) and already partially implemented in `frontend/src/lib/styles/grifel.css` and `GrifelStatBlock.svelte` / `combat/[id]/+page.svelte`.

- **Theme**: hybrid, intentional — chrome (app bar, page background) is dark wood/blood-maroon; content surfaces (cards, panels, lists) are warm parchment. Not full light or full dark; the contrast between the two *is* the design.
- **Fonts**: 'Cinzel' (display — all-caps labels, headings, nav, badges) paired with 'Spectral' (body serif — running text, stat values). Both loaded via Google Fonts in `grifel.css`. No system sans anywhere in themed surfaces.
- **Palette**: maroon/blood-red accent (`--gr-accent #7a1f1f` and the maroon gradient `--gr-maroon`/`--gr-maroon-dark` for chrome), parchment creams (`--gr-parchment-*`) for surfaces, ink browns (`--gr-ink-*`) for text, gold/cream (`--gr-cream-*`) for chrome text and labels. HP bars use green→amber→red tiers (`--gr-hp-*`). Players get a cool blue (`--gr-player-*`) to read distinctly from monsters at a glance.
- **Chrome pattern**: a 54px maroon gradient app bar with a circular ✦ glyph + "ГРИФЕЛЬ" wordmark, Cinzel uppercase nav tabs (active tab = parchment pill with maroon text), right-aligned status/utility controls.
- **Content pattern**: parchment list rows/cards with soft borders (`--gr-parchment-border*`), Cinzel uppercase micro-labels over Spectral values, pill-shaped buttons/badges, rounded ability-score tiles in a 6-col grid for stat blocks.
- **Anti-reference**: no generic SaaS dark-mode-with-purple-accent (the pre-existing un-themed pages use `#222`/`#7c3aed` — that's exactly what's being replaced).

### Design Principles
1. Every themed surface uses the `--gr-*` tokens from `grifel.css`, not ad hoc hex values — extend the token set rather than hardcoding new colors.
2. Cinzel is reserved for short labels/headings/nav/badges (uppercase, letter-spaced); Spectral carries all running text and form content. Never set long body copy in Cinzel or in uppercase.
3. Preserve the maroon-chrome / parchment-content split everywhere — don't let chrome bleed into content panels or vice versa.
4. Reuse the established component vocabulary (pill buttons, parchment cards, Cinzel micro-labels, HP-tier bars, player/monster color coding) instead of inventing new patterns per page.
5. This is a faithful implementation of a specific human-authored reference design, not a from-scratch AI aesthetic — match the reference's existing choices (including its side-accent recharge callout) rather than substituting generic "safe" patterns.
